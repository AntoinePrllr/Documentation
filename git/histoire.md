# L'histoire de Git

## Pourquoi Git a été créé ?

Au début des années 2000, le développement du noyau Linux impliquait des milliers de développeurs répartis dans le monde entier.

Pour gérer toutes les modifications du code, les équipes utilisaient différents systèmes de gestion de versions. Cependant, les solutions existantes présentaient plusieurs limites :

- Certaines étaient payantes
- D'autres étaient lentes
- Elles géraient mal les très gros projets
- Elles dépendaient souvent d'un serveur central

En 2005, la communauté Linux perd l'accès à l'outil qu'elle utilisait alors : BitKeeper.

Face à cette situation, le créateur de Linux décide de développer un nouvel outil.

---

## Linus Torvalds

Git a été créé en 2005 par Linus Torvalds, également créateur du système d'exploitation Linux.

Son objectif était de concevoir un système :

- Rapide
- Gratuit
- Fiable
- Distribué
- Capable de gérer un très grand nombre de contributions

En seulement quelques jours, une première version fonctionnelle de Git voit le jour.

---

## Pourquoi le nom "Git" ?

Le mot "git" est un terme d'argot britannique qui peut être traduit de manière familière par :

> idiot, imbécile ou tête de mule

Linus Torvalds a lui-même expliqué avoir choisi ce nom avec beaucoup d'humour.

Selon lui :

> "Je suis un sale type égoïste, donc je donne mon nom à tous mes projets."

Il a également plaisanté en expliquant que Git pouvait signifier :

- Global Information Tracker
- Goddamn Idiotic Truckload of sh*t

Même si ces significations sont souvent citées, Git n'est pas réellement un acronyme.

---

## Une approche différente

Contrairement aux anciens systèmes de gestion de versions, Git est distribué.

Cela signifie que chaque développeur possède une copie complète du projet sur son ordinateur.

Chaque copie contient :

- Les fichiers du projet
- L'historique complet des modifications
- Les branches
- Les commits

Ainsi, même sans connexion Internet, il est possible de travailler normalement.

---

## L'arrivée de GitHub

En 2008, une nouvelle plateforme voit le jour : GitHub.

GitHub permet :

- D'héberger des dépôts Git en ligne
- De collaborer facilement avec d'autres développeurs
- De partager du code publiquement
- De gérer des projets open source

GitHub n'est pas Git.

Git est l'outil de gestion de versions.

GitHub est un service en ligne utilisant Git.

On peut utiliser Git sans GitHub, et inversement GitHub repose sur Git pour fonctionner.

---

## Git aujourd'hui

Git est devenu le standard mondial de la gestion de versions.

Il est utilisé par :

- Les développeurs indépendants
- Les startups
- Les grandes entreprises
- Les projets open source
- Les équipes de développement de logiciels, de sites web et d'applications

Des millions de projets utilisent Git quotidiennement.

# À retenir

- Git a été créé en 2005 par Linus Torvalds.
- Il a été conçu pour gérer le développement du noyau Linux.
- Git est un système de gestion de versions distribué.
- GitHub est une plateforme utilisant Git, mais ce n'est pas Git lui-même.
- Aujourd'hui, Git est devenu la référence mondiale pour le versionnement du code.
